# Secretscan CLI

`cmd/secretscan` scans guest-visible text files for secret-like values before
an agent packs, summarizes, or shares local context. Findings are always
redacted in the report.

The fixture contains fake token shapes for OpenAI-style keys, GitHub tokens,
AWS access key IDs, JWT-like values, and private key headers.

## CLI Help

The option reference is generated by `@argparse`, so the checked transcript
stays in sync with the command implementation.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/secretscan -- --help
Usage: secretscan [options] [path]

Scan WASI-visible text files for secret-like values before sharing context.

Arguments:
  path  root file or directory [default: .]

Options:
  -h, --help               Show help information.
  -V, --version            Show version information.
  --json                   write compact JSON instead of Markdown
  --hidden                 include hidden entries; default is to skip names starting with .
  --no-default-ignore      disable built-in ignores such as .git, _build, and node_modules
  --all-files              try every readable text file instead of filtering by extension
  -e, --ext <ext>          comma-separated extensions to include, for example env,json,mbt,md
  --max-files <max-files>  maximum number of files to scan [default: 128]
  --max-bytes <max-bytes>  maximum bytes to inspect per file after reading text [default: 200000]
  --limit <limit>          maximum findings to print [default: 80]
  --fail-on <fail-on>      exit with status 1 when findings reach low, medium, high, or critical [default: none]
  -o, --output <output>    write scan report to a guest-visible file
```

## Bundled Skill Artifact

The `portable-secretscan` skill commits a release Wasm artifact, so an agent can
run the scanner through `moonrun` without a MoonBit build step.

```mooncram
$ root="$TESTDIR/../.."; (cd "$root" && moonrun skills/portable-secretscan/assets/secretscan.wasm --limit 2 tests/cram/fixtures/secrets-demo)
# Secret Scan

- root: `tests/cram/fixtures/secrets-demo`
- files scanned: 2
- files skipped: 0
- bytes scanned: 204
- findings: 2
- highest severity: critical
- fail on: none

## Findings

| severity | rule | file | line | evidence |
|---|---|---|---:|---|
| critical | github token | `tests/cram/fixtures/secrets-demo/token.txt` | 1 | `ghp_...qrst (41 chars)` |
| medium | secret assignment | `tests/cram/fixtures/secrets-demo/token.txt` | 1 | `github_token=ghp_...qrst (41 chars)` |

## Notes

Findings are heuristic and redacted. Review the source file before sharing repository context.
```

`--fail-on` is most useful under `moonrun`, because the portable runner
preserves the WASI exit status.

```mooncram
$ root="$TESTDIR/../.."; mkdir -p "$root/.tmp"; (cd "$root" && moonrun skills/portable-secretscan/assets/secretscan.wasm --fail-on high --json tests/cram/fixtures/secrets-demo > .tmp/secretscan-fail.json); printf 'exit=%s\n' "$?"; rm -f "$root/.tmp/secretscan-fail.json"
exit=1
```
