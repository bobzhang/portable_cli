# Svgcheck CLI

`cmd/svgcheck` inspects SVG assets with a lightweight portable scanner. It is
intended for agent workflows that need a quick inventory before embedding,
reviewing, or sharing SVG files.

The examples use checked fixtures under `tests/cram/fixtures/svg` and run the
Wasm target through `moon run --target wasm`.

## CLI Help

The option reference is generated by `@argparse`, so this documentation stays
in sync with the command implementation.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/svgcheck -- --help
Usage: svgcheck [options]

Inspect SVG structure and risky features as a portable WASIp1 CLI.

Options:
  -h, --help             Show help information.
  -V, --version          Show version information.
  --json                 write compact JSON instead of Markdown
  -f, --file <file>      input SVG file; stdin is used when omitted
  -o, --output <output>  write output to a guest-visible file
  --limit <limit>        maximum tag counts and issues to include [default: 40]
  --fail-on <fail-on>    exit non-zero when issues at or above this severity are found: high, medium, low, or none [default: none]
```

## Risk Report

The default report includes SVG dimensions, common element counts, and risky
features such as scripts, event handlers, `foreignObject`, and external
references.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/svgcheck -- --file tests/cram/fixtures/svg/risky.svg
# SVG Check
```

## Failure Threshold

Use `--fail-on` in automated checks when risky SVG features should stop the
workflow.


## Bundled Skill Artifact

The `portable-svg` skill commits a release Wasm artifact, so an agent can run
SVG inspection through `moonrun` without a MoonBit build step.
