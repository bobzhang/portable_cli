# Tomlskill CLI

`cmd/tomlskill` inspects simple TOML-style config files without a native TOML
toolchain. It is useful for agent workflows that need a quick inventory of
tables, keys, value kinds, duplicate sections, or a normalized config view.

The examples use checked fixtures under `tests/cram/fixtures/toml` and run the
Wasm target through `moon run --target wasm`.

## CLI Help

The option reference is generated by `@argparse`, so this documentation stays
in sync with the command implementation.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/tomlskill -- --help
Usage: tomlskill [options] [toml...]

Inspect and normalize simple TOML-style config as a portable WASIp1 CLI.

Arguments:
  toml...  TOML words; stdin is used when omitted

Options:
  -h, --help               Show help information.
  -V, --version            Show version information.
  --json                   write compact JSON instead of Markdown
  --format                 write normalized simple TOML instead of an inspection report
  --raw                    with --get, write only the raw value
  -f, --file <file>        input TOML file; stdin is used when omitted
  -o, --output <output>    write output to a guest-visible file
  --get <get>              dotted key path to extract, such as package.name
  -s, --section <section>  report section to print: summary, tables, keys, or issues
  --limit <limit>          maximum rows to include per section [default: 40]
```

## Value Extraction

Use `--get` for dotted paths and `--raw` when a caller wants only the value.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/tomlskill -- --file tests/cram/fixtures/toml/project.toml --get dependencies.miniio
dependencies.miniio = "0.2.0"
```

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/tomlskill -- --file tests/cram/fixtures/toml/project.toml --get package.description --raw
"Portable skills"
```

`--json` with `--get` gives a compact machine-readable lookup result.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/tomlskill -- --file tests/cram/fixtures/toml/project.toml --get package.description --json
{"found":true,"kind":"string","line":6,"path":"package.description","value":"\"Portable skills\""}
```

## Normalize Simple TOML

`--format` removes comments and blank-line churn while preserving table and key
order for simple TOML files.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/tomlskill -- --file tests/cram/fixtures/toml/project.toml --format
name = "portable_cli"
version = "0.1.0"

[package]
description = "Portable skills"
license = "Apache-2.0"
enabled = true
version = "shadow"

[dependencies]
miniio = "0.2.0"
html_parser = "0.1.7"

[[bin]]
name = "htmlfmt"
path = "cmd/htmlfmt"

[[bin]]
name = "tomlskill"
path = "cmd/tomlskill"

[package]
duplicate = true
empty = 
```

## File Output

The portable I/O surface can write normalized config to guest-visible paths.

```mooncram
$ root="$TESTDIR/../.."; out=".tmp/moon-cram-tomlskill-out.toml"; rm -f "$root/$out"; moon -C "$root" run --target wasm cmd/tomlskill -- --file tests/cram/fixtures/toml/project.toml --section issues --output "$out"; cat "$root/$out"; rm -f "$root/$out"
## Issues

| severity | line | message |
|---|---|---|
| medium | 23 | duplicate table: package first seen on line 5 |
| low | 26 | unrecognized non-empty line |
| medium | 27 | empty value for key: empty |
```

## Bundled Skill Artifact

The `portable-toml` skill commits a release Wasm artifact, so an agent can run
TOML inspection through `moonrun` without a MoonBit build step.

```mooncram
$ root="$TESTDIR/../.."; (cd "$root" && moonrun skills/portable-toml/assets/tomlskill.wasm --file tests/cram/fixtures/toml/project.toml --get dependencies.miniio --raw)
"0.2.0"
```
