# Datascout CLI

`cmd/datascout` turns CSV/TSV-like files into bounded, agent-readable schema and
sample reports. It is intentionally small and portable: no native CSV library,
no filesystem APIs beyond `miniio`, and deterministic output for Cram docs.

## CLI Help

The option reference is generated by `@argparse`, so the checked transcript
stays in sync with the command implementation.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/datascout -- --help
Usage: datascout [options]

Summarize CSV/TSV-like data into bounded Markdown or JSON.

Options:
  -h, --help                   Show help information.
  -V, --version                Show version information.
  --json                       write compact JSON instead of Markdown
  --no-header                  treat the first row as data and synthesize column names
  -f, --file <file>            input delimited file; stdin is used when omitted
  -d, --delimiter <delimiter>  delimiter: auto, comma, tab, semicolon, or a single character [default: auto]
  --max-rows <max-rows>        maximum data rows to scan [default: 1000]
  -s, --sample <sample>        sample rows to include in the report [default: 5]
  -o, --output <output>        write report to a guest-visible file
```

```mooncram
- rows scanned: 3
```

## Bundled Skill Artifact

The `portable-datascout` skill commits a release Wasm artifact, so an agent can
profile a local data file through `moonrun` without a MoonBit build step.

```mooncram
$ root="$TESTDIR/../.."; (cd "$root" && moonrun skills/portable-datascout/assets/datascout.wasm --sample 2 --file tests/cram/fixtures/data/customers.csv)
```
