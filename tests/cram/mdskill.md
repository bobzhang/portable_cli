# Mdskill CLI

`cmd/mdskill` inspects Markdown structure without a host Markdown runtime. It is
useful when an agent needs a quick outline, link inventory, task list, code
fence map, front matter summary, or structural issue list from a local file.

The examples use checked fixtures under `tests/cram/fixtures/markdown` and run
the Wasm target through `moon run --target wasm`.

## CLI Help

The option reference is generated by `@argparse`, so this documentation stays
in sync with the command implementation.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/mdskill -- --help
Usage: mdskill [options] [markdown...]

Inspect Markdown structure as a portable WASIp1 CLI.

Arguments:
  markdown...  Markdown words; stdin is used when omitted

Options:
  -h, --help               Show help information.
  -V, --version            Show version information.
  --json                   write compact JSON instead of Markdown
  -f, --file <file>        input Markdown file; stdin is used when omitted
  -o, --output <output>    write output to a guest-visible file
  -s, --section <section>  Markdown section to print: summary, frontmatter, outline, links, tasks, code, or issues
  --limit <limit>          maximum rows to include per section [default: 40]
```

## Full Report

The default report includes a compact summary followed by front matter, outline,
links, tasks, code fences, and structural issues.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/mdskill -- --file tests/cram/fixtures/markdown/agent-note.md
# Markdown Skill

- lines: 29
- words: 34
- chars: 342
- headings: 5
- links: 2
- tasks: 2 (1 done)
- code fences: 1
- issues: 2

## Front Matter

| key | value |
|---|---|
| title | Portable Markdown |
| owner | agent |

## Outline

- L6: # Portable Markdown (`#portable-markdown`)
- L10: ## Links (`#links`)
- L14: ## Tasks (`#tasks`)
- L19: ## Code (`#code`)
- L27: ## Links (`#links`)

## Links

| line | text | target |
|---|---|---|
| 12 | MoonBit | https://www.moonbitlang.com |
| 12 | empty |  |

## Tasks

| state | line | text |
|---|---|---|
| open | 16 | Add Cram docs |
| done | 17 | Build Wasm artifact |

## Code Fences

| language | start | end | lines |
|---|---|---|---|
| moonbit | 21 | 25 | 3 |

## Issues

| severity | line | message |
|---|---|---|
| medium | 27 | duplicate heading slug: links |
| medium | 12 | link target is empty |
```

## Focused Sections

Use `--section` to print only one documentation slice. This is useful for
smaller agent prompts.

```mooncram
$ moon -C "$TESTDIR/../.." run --target wasm cmd/mdskill -- --file tests/cram/fixtures/markdown/agent-note.md --section outline
## Outline

- L6: # Portable Markdown (`#portable-markdown`)
- L10: ## Links (`#links`)
- L14: ## Tasks (`#tasks`)
- L19: ## Code (`#code`)
- L27: ## Links (`#links`)
```

## File Output

The portable I/O surface can write reports to guest-visible paths.

```mooncram
$ root="$TESTDIR/../.."; out=".tmp/moon-cram-mdskill-out.md"; rm -f "$root/$out"; moon -C "$root" run --target wasm cmd/mdskill -- --file tests/cram/fixtures/markdown/agent-note.md --section issues --output "$out"; cat "$root/$out"; rm -f "$root/$out"
## Issues

| severity | line | message |
|---|---|---|
| medium | 27 | duplicate heading slug: links |
| medium | 12 | link target is empty |
```

## Bundled Skill Artifact

The `portable-markdown` skill commits a release Wasm artifact, so an agent can
run Markdown analysis through `moonrun` without a MoonBit build step.

```mooncram
$ root="$TESTDIR/../.."; (cd "$root" && moonrun skills/portable-markdown/assets/mdskill.wasm --file tests/cram/fixtures/markdown/agent-note.md --section tasks)
## Tasks

| state | line | text |
|---|---|---|
| open | 16 | Add Cram docs |
| done | 17 | Build Wasm artifact |
```
